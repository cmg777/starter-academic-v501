# 2026-06-11 — Batch slide decks, wave 1 (12 tutorials)

First wave of the **batch slides plan**: drive the `write-slides` skill across the
tutorial back-catalog, mirroring the 2026-05-22 write-app batch. Every eligible post
(the 60 with a `web_app/`) is in scope; this wave shipped **12** decks. After this wave:
**14 of 60 posts have a Quarto reveal.js deck** (2 prior — `r_double_lasso` reference +
`python_did_sc_tsunami` — plus these 12); **46 remain**.

## Method

12 posts (4 Python / 4 R / 4 Stata) generated by **parallel subagents** (one validation
wave of 4 spanning all three languages, then a fan-out of 8). Each agent executed the
`write-slides` SKILL.md **manually** in autonomous **`--no-verify`** mode: read the post +
any `results_report.md`, self-answered the interview with **Teaching-audience** defaults,
copied `site-brand.scss` + `title-slide.html` **verbatim**, authored `slides.qmd` from the
template against the `r_double_lasso/slides/slides.qmd` exemplar, ran
`/Applications/quarto/bin/quarto render` → `index.html` + `slides_files/`, and injected the
`Slides (HTML)` button as the first `links:` entry. No interview prompts, no ES/JA copies,
no commits by the agents.

## Key decisions & learnings

- **Icon fix baked in.** Used `icon: chalkboard-teacher` / `icon_pack: fas` (FA5), **not**
  the skill's documented `person-chalkboard` (FA6) — the FA6 name does not render in this
  site's Font Awesome build (per `7f292bc`, which hand-fixed the tsunami deck). Label kept
  as `"Slides (HTML)"`, relative `url: slides/index.html`, no trailing slash.
  **Follow-up:** the `write-slides` SKILL.md (Phase 3.5) and the `r_double_lasso` reference
  deck still carry `person-chalkboard` — update both for consistency.
- **Figure prefixes vary by post** (`did101_*`, `doubleml_*`, `fwl_*`, `pca_*`, …), not
  always `<slug>_*`. Agents detected the real filenames via `ls` and referenced
  `../<actual>.png`; a post-batch check confirmed **all 62 figure references resolve on
  disk** (0 missing).
- **`--no-verify` quality gate.** Static integrity verified across all 12: `index.html`
  48–64 KB, **0** leftover `{{…}}` placeholders, **0** executable code fences (decks are
  illustrative `{.r}`/`{.python}`/`{.stata}`, so render is pure Pandoc — no kernel), **0**
  `html-math-method: katex` mis-settings, link + icon injected on each. **Not** checked:
  visual math rendering (the skill's Playwright Layer-C) — low risk since the decks copy the
  proven MathJax config of the two reference decks, but a manual eyeball remains the one open
  item.
- **Rendered output committed.** ~97 MB total (12 × ~8 MB, 119 files/deck = sources +
  `index.html` + `slides_files/`). These are production assets Netlify serves (Netlify runs
  Hugo, not Quarto), so they live in git; only `content/post/*/slides/.quarto/` is ignored.
- All 12 rendered clean on the **first** `quarto render` pass (1.8.27) — no render-and-fix
  loop needed.
