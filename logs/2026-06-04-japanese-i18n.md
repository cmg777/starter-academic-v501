# Japanese (/ja/) added as the third site language

**Date:** 2026-06-04
**Branch/commit:** `master` (shipped live, per request — same as how Spanish shipped).

## TL;DR
Added **Japanese at `/ja/`**, mirroring the Spanish (`/es/`) footprint exactly. The i18n
infrastructure had been pre-staged when Spanish was built, so this was (1) activating six
infra hooks and (2) creating `content/ja/` with Japanese translations (です・ます polite
register). Verified with a clean local Hugo 0.111.3 build: **EN 862 / ES 139 / JA 139
pages**, identical structure (26 homepage sections each for ES and JA).

## What changed

### Infrastructure (6 hooks, all previously pre-staged)
- `config/_default/config.yaml` — `hasCJKLanguage: true`; en mount `excludeFiles:
  '{es,ja}/**'`; added the `content/ja` mount (`lang: ja`).
- `config/_default/languages.yaml` — replaced the commented `ja:` block with a full `ja`
  language (code `ja-jp`, name `日本語`, weight 3, JP `params.description`, and a full
  Japanese `menu.main` mirroring the Spanish menu — **Courses omitted** to avoid a `/ja/`
  404).
- `netlify/edge-functions/geo-lang.ts` — activated `JP: "/ja/"` in `COUNTRY_TO_LANG` and
  added `"ja"` to `KNOWN_LANGS`.
- `assets/js/lang-pref.js` — `KNOWN = { es: 1, ja: 1 }`.
- No template/partial changes — `tutorial_card.html` (`card_url`) and `custom_head.html`
  (hreflang) are already language-agnostic.

### Content — `content/ja/` (mirror of `content/es/`)
Scaffolded with `cp -R content/es content/ja` (brings all images + `cite.bib` along), then
each `.md` translated in place:
- **home/** (11 widgets + gallery) — hero, about, featured, talks, projects, posts, people,
  researchLab (incl. the GEE-map quiz), contact, alumni-link. Hero rotator lead set to
  「研究テーマ：」.
- **authors/** (13) — admin + 12 students. Translated bio/role/interests/education;
  romanized names kept with Japanese country (e.g. `Bimo Arvianto（インドネシア）`); social
  links / emails / org URLs unchanged. `user_groups` translated to **博士課程学生 /
  修士課程学生**, kept byte-identical to `content/ja/home/people.md` (query keys).
- **publication/** (3), **event/** (5), **projects/** (5) — full prose translation
  (title/summary/abstract/body), preserving every `<iframe>`, Canva/YouTube embed,
  `fullwidth-iframe` shortcode, `<details>` dashboard wrapper, link `name:` labels, and
  English `tags`/`projects` query keys. (These three sections were translated by three
  parallel translator subagents under a shared glossary; spot-checked for preserved
  shortcodes/iframes and English tags.)
- **post/** (6) — card stubs: translated `title` + `summary` only; kept `card_url`,
  English `categories`, `_build: {render: never, list: always}`. Bodies stay empty; cards
  link to the English tutorials.

## Geolocation
Japan visitors are auto-redirected `/ → /ja/` on first homepage visit (mirrors the Spanish
behavior): once per browser (`geo_seen`), manual switcher choice (`lang_pref`) always wins.
Testable only on a Netlify deploy preview / `netlify dev`, e.g.
`curl -H 'x-nf-geo: {"country":{"code":"JP"}}' http://localhost:8888/` → 302 `/ja/`.
