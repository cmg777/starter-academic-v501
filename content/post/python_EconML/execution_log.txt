STEP 2b: NAIVE COMPARISON (why we need causal ML)
======================================================================

  Simple difference-in-means (no confounder adjustment):
  Comparison         Naive   Ground Truth     Bias
  -----------------------------------------------
  1-0                0.109          0.250   -0.141
  2-1                0.098          0.050   +0.048
  3-1                0.413          0.300   +0.113

  The naive 1-0 estimate is biased because mining districts
  differ systematically from non-mining districts (geography,
  institutions). The Causal Forest controls for these confounders
  via the Double Machine Learning residualization step.

======================================================================
STEP 3: CAUSAL FOREST DML — NTL
  Outcome: ntl_log  |  Treatment: treatment (M=4)
  Observations: 3,000  |  Trees: 500
======================================================================
  Configuring CausalForestDML...
    - discrete_treatment=True (4 treatment levels)
    - DML: GradientBoosting for both nuisance models
    - 500 honest causal trees (separate split/estimation samples)
    - BLB inference (bootstrap of little bags)
    - GroupKFold CV by district_id (prevents within-district
      data leakage in cross-fitting; does NOT cluster SEs)
  Fitting (first-stage residualization + causal forest)...
  Done in 0.4 minutes
