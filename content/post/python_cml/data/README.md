# Data dictionary - Causal Machine Learning for Policy Evaluation

Companion data for a hands-on Python tutorial on <strong>Causal Machine Learning (CML)</strong> for active-labour-market-programme (ALMP) evaluation. The cohort is a <strong>fully synthetic</strong> Flemish-ALMP-style sample of 5,000 jobseekers with six pre-treatment covariates, a binary training indicator, and months employed over a 30-month follow-up. The post walks the full CML roadmap &mdash; the average treatment effect (<strong>ATE</strong>) via DoubleML's cross-fitted, doubly-robust Interactive Regression Model; group effects (<strong>GATE</strong>) by Dutch proficiency via doubly-robust pseudo-outcome averaging; individual effects (<strong>IATE</strong>) via EconML's CausalForestDML; and a welfare-maximising training-assignment rule. Because the data are simulated, the <em>true</em> individual treatment effect of every jobseeker is known, so every estimator is benchmarked against ground truth. The data-generating process is modelled on Cockx, Lechner &amp; Bollens (2023) and the methodological roadmap in Lechner (2023).


Companion data for the post <https://carlos-mendez.org/post/python_cml/>. Each dataset ships as a labeled Stata `.dta` (v118) plus its source file. Generated by `build_data_dictionary.py`.

## Known limitations & caveats

- <strong>Synthetic data.</strong> There is no real cohort behind this tutorial; results are internally consistent with the calibration but are not empirical evidence about any real ALMP.
- <strong>Truth columns are not observable predictors.</strong> <code>Y0</code>, <code>Y1</code>, <code>tau</code>, and <code>pi_true</code> in <code>cml_truth.csv</code> are knowable only because the data are simulated. They are for benchmarking and welfare scoring only — feeding any of them into an estimator is circular and must not be done.
- <strong>Positional join.</strong> <code>cml_data.csv</code> and <code>cml_truth.csv</code> share no key column; row <em>i</em> of one matches row <em>i</em> of the other. Preserve row order when merging (e.g. <code>pd.concat([df, truth], axis=1)</code>), never sort one file independently.
- <strong>Easy overlap by construction.</strong> True propensities are clipped to [0.05, 0.95] (and the logistic-regression estimate lands inside [0.21, 0.81]), so trimming choices and doubly-robust denominators are not stressed here. Real ALMP cohorts have tighter overlap and trimming matters more.
- <strong>Treatment share is high (≈53%).</strong> Calibrated to keep overlap comfortable in every Dutch-proficiency stratum; do not over-interpret the magnitude of effects as representative of a real programme.
- <strong>Unconfoundedness holds by construction.</strong> The DGP satisfies selection-on-observables over the six covariates; in a real application this is the strong identifying assumption that justifies DoubleML and CausalForestDML over a naive comparison.
