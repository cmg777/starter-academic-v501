# Data dictionary - Introduction to PCA for Building Development Indicators

Companion data for a hands-on Python tutorial that builds a composite <strong>Health Index</strong> from two correlated health indicators using Principal Component Analysis (PCA). The dataset is <strong>fully simulated</strong>: 50 countries are generated from a single latent health factor that drives <code>life_exp</code> (Life Expectancy) positively and <code>infant_mort</code> (Infant Mortality) negatively, so that PCA's recovery of the known single-factor structure can be verified. The post follows a six-step manual pipeline in NumPy/pandas &mdash; polarity adjustment, z-score standardization, the covariance matrix, eigen-decomposition, PC1 scoring, and Min-Max normalization &mdash; then replicates it with scikit-learn. Polarity adjustment flips the raw correlation from &minus;0.9595 to +0.9595; eigen-decomposition yields eigenvalues 1.9595 and 0.0405 with equal weights of 0.7071, so PC1 explains 97.97% of total variance and the manual scores match scikit-learn to within 1.33&times;10&#8315;&#185;&#8309;. This file is the single raw input the whole pipeline is built on.


Companion data for the post <https://carlos-mendez.org/post/python_pca/>. Each dataset ships as a labeled Stata `.dta` (v118) plus its source file. Generated by `build_data_dictionary.py`.

## Known limitations & caveats

- <strong>Synthetic data.</strong> There is no real data behind this tutorial; values are drawn from a known single-latent-factor process and are internally consistent with that calibration, but are not empirical evidence about real-world health outcomes.
- <strong>Two standardized variables → equal weights.</strong> With exactly two standardized indicators, PCA always assigns equal weights (1/&radic;2 each) regardless of the correlation, so PC1 is just a (scaled) simple average. PCA's real advantage appears only with three or more variables.
- <strong>Relative, not absolute.</strong> The Health Index measures performance relative to the 50-country sample (Min-Max maps the worst to 0 and the best to 1); it is not an absolute or cross-sample-comparable score.
- <strong>Derived columns are not stored here.</strong> <code>infant_mort_adj</code>, z-scores, PC1, and the Health Index are computed in the script and saved to the post's results CSVs; this file holds only the three raw input columns.
