# Data dictionary - Causal Inference with DoWhy: Working From Home & Productivity

Companion data for a beginner-friendly causal-inference tutorial that uses <strong>DoWhy</strong>'s four-step framework (Model, Identify, Estimate, Refute) to recover a <strong>known</strong> average treatment effect from observational data. The dataset is <strong>fully synthetic</strong>: 5,000 employees with a binary work-from-home treatment, a continuous productivity outcome, two confounders (introversion, number of children) that drive self-selection into treatment, and a subway-disruption instrument that satisfies the exclusion restriction by construction. Because the true causal effect is fixed at <strong>1.0 productivity points</strong>, every estimator can be scored against the truth: the naive difference in means overshoots to 1.39, while the backdoor methods (regression, IPW, doubly robust) recover ~1.0 and IV (2SLS) returns 0.89 with a much wider interval. The entire data-generating process is open and reproducible.


Companion data for the post <https://carlos-mendez.org/post/python_dowhy_intro/>. Each dataset ships as a labeled Stata `.dta` (v118) plus its source file. Generated by `build_data_dictionary.py`.

## Known limitations & caveats

- <strong>Synthetic data.</strong> There is no real survey behind this dataset; values are drawn from the data-generating process above. Results are internally consistent with the simulation but are not empirical evidence about real-world work-from-home effects.
- <strong>Known truth by design.</strong> The true ATE is fixed at 1.0 so methods can be scored. On real data the truth is unknown — that is exactly why identification (DoWhy's Step 2) and refutation (Step 4) matter.
- <strong>Cross-section, not a panel.</strong> Each row is one simulated employee at a single point; there is no time dimension, no repeated measurement, and no row identifier.
- <strong>Exclusion restriction is assumed true here.</strong> <code>subway_disruption</code> is built to affect productivity only through WFH choice. In practice the exclusion restriction cannot be tested from data and must be justified by domain knowledge.
- <strong>Selection on observables is satisfiable here.</strong> The two confounders are fully observed, so backdoor methods are unbiased by construction; with a missing confounder they would be biased while IV would remain valid.
