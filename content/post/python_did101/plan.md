# Plan: Python DiD101 Tutorial using PyFixest and Great Tables

## Context

The Stata DiD tutorial (`content/post/stata_did/index.md`, 1,208 lines) introduces Difference-in-Differences through a case study of an after-school tutoring program in 10 of 35 schools. The user wants a Python counterpart at `content/post/python_did101/` that uses the **same case study and datasets** but showcases PyFixest's estimation, inference, and reporting capabilities alongside Great Tables for publication-quality output.

The Python version should slightly differ from the Stata version to highlight PyFixest strengths: unified formula syntax (`"Y ~ X | FE"`), stepwise specifications (`csw0()`), flexible inference (iid/HC1/CRV1/CRV3), tidy output methods (`.coef()`, `.se()`, `.tidy()`), `etable()` with Great Tables integration, and built-in event study visualization (`iplot()`).

### Section 7: DiD via Regression (~200 lines)
The core section showcasing PyFixest.

**7.1: Classical OLS Interaction**
- **Equation:** Classical DiD regression (display math)
- Code: `fit_ols = pf.feols("gpa ~ treated + post + txp", data=df, vcov="HC1")`
- Output: `fit_ols.summary()`
- Interpretation: constant=71.22, treated=−11.05, post=10.89, txp=25.31
- **PyFixest features:** `feols()`, `vcov="HC1"`, `.summary()`

**7.2: TWFE with Fixed Effects**
- **Equation:** TWFE model (display math)
- Explain `|` pipe syntax for absorbing FE
- Code: `fit_twfe = pf.feols("gpa ~ txp | id + time", data=df, vcov={"CRV1": "id"})`
- Output + interpretation
- **PyFixest features:** `"Y ~ X | FE1 + FE2"`, `vcov={"CRV1": "id"}`

**7.3: TWFE with Covariate**
- Code: `fit_cov = pf.feols("gpa ~ txp + female_share | id + time", data=df, vcov={"CRV1": "id"})`
- Interpret: estimate shifts 25.31→25.33, female_share insignificant
- **PyFixest features:** Covariates alongside absorbed FE

**7.4: Programmatic Access to Results** *(NEW vs Stata)*

## Potential Issues

1. **`etable()` API may have changed** — PyFixest recently moved to `maketables` package. Fallback: use `etable()` for text, build GT manually from `.tidy()`.
2. **`iplot()` return type** — May render inline rather than returning saveable figure. Fallback: extract coefficients manually, build matplotlib plot.
3. **`timeToTreat` NaN handling** — Event study data has NaN for untreated schools' timeToTreat. PyFixest's `i()` should handle this, but verify.
4. **Exact SE match** — Point estimates will match Stata exactly. SEs may differ slightly due to small-sample corrections (degrees of freedom adjustments).
