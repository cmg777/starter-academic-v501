# Data dictionary - Evaluating a Cash Transfer Program (RCT) with Panel Data

Companion data for a hands-on Stata tutorial that evaluates the causal effect of a cash transfer program on household consumption. The data are <strong>fully synthetic</strong>: 2,000 households in a developing country, observed in a balanced panel across a 2021 baseline and a 2024 endline (4,000 observations). The outcome is log monthly consumption and the program raises it by a <strong>known true effect of 12%</strong> (0.12 log points). The tutorial walks from baseline-balance checks through three cross-sectional estimators &mdash; regression adjustment (RA), inverse probability weighting (IPW), and doubly robust AIPW/IPWRA &mdash; then difference-in-differences and doubly robust DiD on the panel, and an endogenous-treatment IV model for imperfect compliance. Because the ground truth is known, every estimate can be checked against 0.12.


Companion data for the post <https://carlos-mendez.org/post/stata_rct/>. Each dataset ships as a labeled Stata `.dta` (v118) plus its source file. Generated by `build_data_dictionary.py`.

## Known limitations & caveats

- <strong>Synthetic data.</strong> There is no real program behind this tutorial; values are simulated with a known 0.12 log-point effect. Results are internally consistent with the calibration but are not empirical evidence about real cash-transfer programs.
- <strong>Two periods only.</strong> The panel has a single baseline (2021) and endline (2024), so it cannot test pre-treatment (parallel) trends or estimate dynamic/staggered effects.
- <strong>Homogeneous effect by construction.</strong> The treatment effect is the same for all households, so ATE and ATT nearly coincide; real programs typically show heterogeneity worth exploring by subgroup.
- <strong>Imperfect compliance.</strong> Random assignment <code>treat</code> &ne; actual receipt <code>D</code> (85% take-up among the offered, 5% among controls); offer-based (ITT) and receipt-based estimates differ — use <code>treat</code> for the policy offer and the IV/DR-receipt models for the effect of receipt.
- <strong>DGP internals.</strong> <code>alpha</code> and <code>eps</code> are simulation components (household and idiosyncratic terms), and <code>y0</code> is the baseline outcome carried to both rows; they are exposed for transparency and are not analysis covariates in the tutorial.
