# Post resource buttons & tutorial bundles

Front-matter `links:` entries render as resource buttons via `layouts/partials/page_links.html` (pinned Wowchemy module). The key rule that partial enforces: it only adds `target="_blank" rel="noopener"` when a `links:` entry's `url` has an `http(s)` scheme; a **relative** `url` resolves as a page-bundle resource and opens in the **same tab**.

## Slides (PDF) link button

When a post ships a slide deck PDF in its page bundle (`content/post/<slug>/slides.pdf`, served at `/post/<slug>/slides.pdf`), surface it as a resource button.

**Trigger:** the user adds a `slides.pdf` (or says "Add slides to `<post slug>`"). Add ONE entry to the post's front-matter `links:` section — nothing in the body:

```yaml
- icon: file-pdf
  icon_pack: fas
  name: "Slides (PDF)"
  url: https://carlos-mendez.org/post/<slug>/slides.pdf
```

**Use an absolute URL (not relative).** A **relative** `url: slides.pdf` resolves correctly (page-bundle resource) but opens in the **same tab**; the **absolute** `https://carlos-mendez.org/post/<slug>/slides.pdf` opens the deck in a **new tab** (and matches the post's other absolute links like "MD version" / "Data (CSV)"). `file-pdf` + `fas` is the site's standard PDF icon (also used by publications).

**Why not the native `url_slides` field:** Wowchemy's `url_slides` also renders a button (it resolves a relative `slides.pdf` and opens a new tab), but it is hard-labeled **"Slides"** via the site-wide `btn_slides` i18n string — *not* customizable per post. Use `url_slides: slides.pdf` only if a plain "Slides" label is fine; use the `links:` entry above when you want the "Slides (PDF)" label + PDF icon.

- No body change, no embed; the PDF lives in the bundle.
- **No i18n change** — the ES/JA counterparts are card-only stubs (`_build: {render: never}`) that link back to the English post.
- Reference implementation: `content/post/r_sc_multi_country/` (ships `slides.pdf` + the `links:` entry).

## Slides (HTML) link button

When a post ships a Quarto reveal.js deck (`content/post/<slug>/slides/`, served at `/post/<slug>/slides/index.html`), add ONE `links:` entry — nothing in the body:

```yaml
- icon: person-chalkboard
  icon_pack: fas
  name: "Slides (HTML)"
  url: slides/index.html
```

**Use a RELATIVE `url` with no trailing slash.** `url: slides/index.html` resolves as a page-bundle resource and opens in the same tab (identical to the proven `web_app/index.html`). A trailing-slash `url: slides/` is re-rooted by the Wowchemy theme to an absolute `/slides/` and 404s. This is **distinct** from "Slides (PDF)" (root `slides.pdf`, absolute URL, new tab) and from Wowchemy's native `url_slides` — all three can coexist on one post.

The deck is generated by the **`write-slides`** skill — see `.claude/skills/write-slides/SKILL.md` (the `Slides (HTML)` link injection step) and `.claude/skills/write-slides/references/render-and-fix.md` (the trailing-slash bug). No i18n change (English post only); the deck reuses the post's figures via relative `../<slug>_*.png` paths.

## Tutorial bundles (`.zip`)

Some Quarto tutorials ship companion files (`setup_env.py`, `_quarto.yml`, `render.command`/`render.bat`, bundle `README.md`) alongside the `.qmd` so students can extract a ZIP and double-click `render.command` (macOS) or `render.bat` (Windows) for a one-click hermetic render.

**Convention (post-root `<slug>.zip`):** bundles built by `write-quarto-notebook-python` live at `content/post/<slug>/<slug>.zip`, linked from `index.md` with `name: "Quarto project (.zip)"` and `url: <slug>.zip`. Rebuild any of these with `bash content/post/<slug>/build_bundle.sh` after editing the source files. The mechanism is documented in `.claude/skills/write-quarto-notebook-python/SKILL.md` and `.claude/skills/write-quarto-notebook/SKILL.md`.

**Which posts ship a bundle is derivable** — don't hand-maintain a list:
```bash
ls content/post/*/build_bundle.sh    # posts with a bundle build script
ls content/post/*/*.zip              # the built post-root bundles
```

**Legacy exception (`static/uploads/`):** `content/post/python_pyfixest/` predates the post-root convention. Its bundle ships as `static/uploads/python_pyfixest_tutorial.zip`, linked as "Tutorial bundle (.zip)". Whenever `tutorial.qmd`, `setup_env.py`, `_quarto.yml`, `script.py`, or the bundle `README.md` changes, run `bash content/post/python_pyfixest/build_bundle.sh` and commit the regenerated zip in the same commit. Can be migrated to the post-root convention as a separate cleanup task.
